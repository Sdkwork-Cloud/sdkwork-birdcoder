# 15-工作台偏好-外部Terminal集成-本地存储补充标准

## 1. 目标

统一 BirdCoder 在 `Code / Studio / Settings` 的本地数据边界，并明确外部 Terminal 集成只保存“接入偏好与引用”，不保存终端 authority 事实。

## 2. 本仓 authority 本地数据

- `workbench_preference`
- `workspace`
- `project`
- `coding_session_*`
- `run_configuration`
- `build / preview / simulator / test` 证据与摘要
- `appbase` 投影数据

说明：

- 上述数据由本仓管理、迁移、审计、回放。
- 外部 Terminal 的真实会话、输出、命令历史、恢复状态不进入本仓 authority。

## 3. 外部 Terminal 集成只允许保存的本地偏好

- `externalTerminalProviderId`
- `externalTerminalProfileId`
- `externalTerminalLaunchMode`
- `lastExternalTerminalTargetId`
- `lastExternalTerminalWorkspaceBinding`
- `lastExternalTerminalProjectBinding`

这些字段只表达“如何接入外部 Terminal”，不表达“外部 Terminal 内部发生了什么”。

## 4. 禁止项

- 不在本仓本地数据库中沉淀 `stdout / stderr / command history / pty buffer` authority 数据。
- 不在 `localStorage` 中缓存完整 terminal session/execution 事实。
- 不把外部 Terminal 会话恢复、命令回放、输出归档拉回本仓重建一套私有模型。
- 已存在的 `terminal_session`、`terminal_execution` 占位结构不得继续扩张为产品主标准；若仍保留，仅视为兼容占位或待退役遗留。

## 5. 宿主责任

| 宿主 | 本仓负责 | 外部 Terminal 工程负责 |
| --- | --- | --- |
| Desktop | 启动映射、上下文注入、状态摘要引用 | 真正终端运行时、PTY、CLI 执行 |
| Web | 外部入口、跳转、远程接入映射 | 真实终端执行与会话管理 |
| Server | 集成协议、审计摘要、发布接口 | 终端服务本体与运行态 authority |

## 6. 对齐标准

- `Code` 与 `Studio` 共用同一份工作台偏好。
- 外部 Terminal 只读取统一 `run_configuration` 与上下文绑定。
- Terminal 集成配置进入 Settings，但 Terminal 内部事实不进入本仓 authority。

## 7. 评估标准

- 数据边界是否清晰：本仓是否只保存偏好与引用。
- 主线聚焦是否正确：本地 authority 是否继续围绕 `Code / Studio / coding_session / Studio evidence`。
- 集成成本是否可控：替换外部 Terminal 工程时，是否无需改本仓核心数据模型。
