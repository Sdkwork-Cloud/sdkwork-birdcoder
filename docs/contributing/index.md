# Contributor Guide

BirdCoder contributors follow the SDKWork architecture standard and package-first discipline.

## Expectations

- Preserve BirdCoder package names and product identity
- Keep delivery modes aligned through shared release scripts and workflows
- Prefer architecture-shape parity over feature copying
- Run `pnpm lint`, `pnpm run check:arch`, and `pnpm docs:build` before closing release-facing work
