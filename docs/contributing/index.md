# Contributor Guide

BirdCoder contributors follow the same package-first architecture discipline as Claw Studio, while keeping BirdCoder business modules product-specific.

## Expectations

- preserve BirdCoder package names and product identity
- keep delivery modes aligned through shared release scripts and workflows
- prefer architecture-shape parity over feature copying
- run `pnpm lint`, `pnpm build`, and `pnpm docs:build` before closing release-facing work
