## Highlights

- Tightens Claw workflow parity by introducing a BirdCoder-side `run-desktop-release-build.mjs` phase wrapper for `sync`, `prepare-target`, `prepare-openclaw`, and `bundle`.
- Adds a Claw-compatible `run-claw-server-build.mjs` alias so reusable release jobs can use the same server build entry shape without changing BirdCoder runtime behavior.
- Upgrades `render-release-notes.mjs` to prefer `docs/release/releases.json` and release markdown entries when a tagged release note exists, while preserving finalized-manifest fallback for local validation tags.
