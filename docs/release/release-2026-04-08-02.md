## Highlights

- Introduces a BirdCoder-side `run-desktop-release-build.mjs` phase wrapper for `sync`, `prepare-target`, `prepare-server`, and `bundle`.
- Adds a `run-server-build.mjs` alias so reusable release jobs can use the same server build entry shape without changing BirdCoder runtime behavior.
- Upgrades `render-release-notes.mjs` to prefer `docs/release/releases.json` and release markdown entries when a tagged release note exists, while preserving finalized-manifest fallback for local validation tags.

## Scope

- `.github/workflows/ci.yml`
- `.github/workflows/release-reusable.yml`
- `scripts/run-desktop-release-build.mjs`
- `scripts/run-server-build.mjs`
- `scripts/release/render-release-notes.mjs`
- `scripts/release-flow-contract.test.mjs`
- `scripts/run-desktop-release-build.test.mjs`

## Verification

- `pnpm run lint`
- `pnpm run build`
- `node scripts/run-vitepress.mjs build docs`
- `node scripts/run-desktop-release-build.mjs --phase sync`
- `node scripts/run-desktop-release-build.mjs --phase prepare-target`

## Notes

- BirdCoder remains an AI IDE and does not bundle third-party runtimes.
- Release evidence remains anchored by `artifacts/release-validation-v6`.
