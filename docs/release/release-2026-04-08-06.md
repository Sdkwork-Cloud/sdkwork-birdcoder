## Highlights

- Completes the Step 08 terminal launch-profile pass by promoting the terminal dropdown to a unified launch-profile source shared from `sdkwork-birdcoder-commons`, instead of page-local profile assembly.
- Splits the terminal profile chooser into `Shells` and `AI CLI`, so Codex, Claude Code, Gemini, and OpenCode follow one discoverable launch surface with the same metadata model.
- Adds desktop PATH-based CLI readiness detection through Tauri, and shows `Ready / Install / Unknown` state badges for AI CLI profiles in the terminal dropdown.
- Extends the terminal registry contract so non-desktop hosts fall back to `unknown` availability while desktop builds compile and pass the multi-mode verification suite.
