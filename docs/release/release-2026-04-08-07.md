## Highlights

- Adds a terminal launch-guard rule so AI CLI profiles with `missing` availability cannot create new tabs or restore recent sessions.
- Moves the launch decision into the shared terminal runtime contract via `resolveTerminalProfileLaunchState`, keeping the guard logic out of page-local ad hoc checks.
- Preserves browser-host flexibility by continuing to allow `unknown` availability, while desktop hosts use Tauri PATH detection to decide whether a CLI is launchable.
- Keeps Step 08 documentation, architecture acceptance criteria, and release records aligned with the new launch-blocking behavior.

## Notes

- This iteration intentionally guards only known-missing desktop CLI profiles. It does not block `unknown` browser-host profiles, because the browser cannot authoritatively inspect local PATH.
