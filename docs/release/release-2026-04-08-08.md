## Highlights

- Extends the shared terminal runtime with a launch-state presentation model so UI surfaces consume one source of truth for AI CLI availability.
- Makes missing AI CLI launch options visually disabled in the terminal dropdown instead of looking identical to runnable entries.
- Marks blocked recent sessions with the same shared status signal, so users can distinguish restorable sessions from ones that require CLI installation first.
- Keeps Step 08 architecture and delivery documents aligned with the new UX rule that missing CLI state must be visible, not only enforced.
