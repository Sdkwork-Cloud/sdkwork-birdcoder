## Highlights

- Upgrades unified Studio evidence export into a diagnostics-ready JSON contract with `laneFilter` and normalized summary metadata.
- Preserves project scope, profile scope, lane scope, and latest launch time in exported evidence slices, making release triage and offline diagnostics faster.
- Updates Step 07 and architecture baselines so export is no longer just raw entry dumping but part of the governed evidence contract.

- Export file names now encode the active lane, for example `studio-evidence-build-<timestamp>.json`.
- The exported summary is intentionally compact: `entryCount`, `lanes`, `projectIds`, `profileIds`, and `latestLaunchedAt`.
- This iteration improves diagnostics/export/triage convergence without coupling viewer exports to release finalization logic.
