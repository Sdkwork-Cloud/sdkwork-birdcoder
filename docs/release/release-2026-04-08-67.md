## Highlights

- Extends Step 10 terminal governance from blocking-only behavior into durable local audit evidence.
- Adds a dedicated terminal governance audit store so blocked dangerous commands persist `traceId`, command, cwd, approval policy, category, and reason in one record.
- Keeps the existing terminal governance gate small by extending the same contract test instead of adding parallel rule drift.

## Notes

- This closes the first durable evidence slice of `CP10-2`; manual approval recovery and cross-host audit aggregation are still carry-forward items.
- The persistence model is intentionally local and bounded so later SQLite, diagnostics, and release evidence flows can reuse one audit record shape without reworking the block semantics.
- The next optimal Step 10 slice is to consume these stored audit records in recovery or diagnostics surfaces, not to introduce a second competing governance store.
