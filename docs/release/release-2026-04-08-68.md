## Highlights

- Extends Step 10 terminal governance from persisted blocked-command audits into a real recovery surface inside Terminal.
- Standardizes recovery action and description helpers so runtime, audit storage, and UI all describe blocked commands with one contract.
- Surfaces recent blocked governance audits in the terminal dropdown, making recovery actionable instead of leaving audit records as passive evidence only.

## Notes

- This closes the first recovery-consumption slice of `CP10-2`; blocked terminal governance events are now visible, explainable, and recoverable from the shared Terminal surface.
- Recovery semantics remain shared-first: `auditStore.ts` owns the record shape and recovery helpers, while `TerminalPage.tsx` only consumes the standardized contract.
- The next optimal Step 10 slice is to reuse the same governance evidence in diagnostics/export or broader tool families, not to create a second recovery model.
