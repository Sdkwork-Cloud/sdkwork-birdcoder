## Highlights

- Extends Step 10 terminal governance from copyable diagnostics into copyable release-note markdown.
- Adds a shared governance release-note template builder so blocked terminal audits can move into release drafting without manual rewriting.
- Exposes a `Copy Release Note` action in Terminal governance recovery, keeping release collaboration on the same audit contract used by runtime, recovery, and diagnostics.

## Notes

- This closes the first release-collaboration slice of `CP10-2`; terminal governance evidence is now blockable, persisted, recoverable, diagnosable, and draftable for release notes.
- The release-note template remains shared-first: `auditStore.ts` owns the markdown assembly while `TerminalPage.tsx` only handles clipboard delivery.
- The next optimal Step 10 slice is to aggregate these governance templates into broader release assets or cross-tool evidence views instead of introducing more Terminal-only variants.
