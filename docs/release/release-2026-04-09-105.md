## Highlights

- Adds an explicit `releaseControl` contract to `release:plan`, `release:finalize`, manifest fallback release notes, and the local release wrapper.
- Standardizes `releaseKind`, `rolloutStage`, `monitoringWindowMinutes`, `rollbackRunbookRef`, and optional `rollbackCommand` so formal, canary, hotfix, and rollback flows share one release vocabulary.
- Updates Step 13, release architecture, quality-gate standards, prompt instructions, and command references so GitHub release operations stay aligned with the same release metadata.
