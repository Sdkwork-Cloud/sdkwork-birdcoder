## Notes

- `release:rollback:plan` is read-only. It does not publish, delete, or mutate release assets; it standardizes rollback planning and evidence lookup inside the repo.
- The local wrapper now tolerates the standalone `--` token that `pnpm` forwards into script arguments, so rollback planning works through the real workspace command and not only through direct `node` invocation.
- As of `2026-04-09`, the only known `check:quality:standard` blocker remains the Windows `toolchain-platform` preflight on `cmd.exe` and `esbuild.exe` `spawn EPERM`.
