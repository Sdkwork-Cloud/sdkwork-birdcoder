## Highlights

- Turns Step 13 post-release observation and writeback into machine-rendered release-note output instead of leaving those actions in human-only release checklists.
- Adds a deterministic rollback entry to rendered notes, falling back to `release:rollback:plan` when no explicit `rollbackCommand` is present.
- Aligns release, step, prompt, and command documentation on the same `Post-release operations` contract so GitHub Release text and repository-owned notes stay operationally identical.
