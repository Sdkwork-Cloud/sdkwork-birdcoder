## Highlights

- Upgrades `sdkwork-birdcoder-appbase` from a page-entry bridge into a registry-and-manifest-aligned appbase bridge.
- Aligns the active architecture docs so `auth/user/vip` are no longer described as standalone BirdCoder business modules.
- Tightens the reusable Step prompt and Step 14 contract so future iterations must preserve `catalog / registry / manifest / route intent / storage scope` as one unified appbase surface.

## Notes

- BirdCoder still uses a local bridge package instead of directly importing sibling `sdkwork-appbase` runtime packages, because the current workspace does not yet carry the full external dependency graph required by those packages.
- The bridge now preserves more of the upstream appbase architecture shape, making a later direct replacement materially easier.
