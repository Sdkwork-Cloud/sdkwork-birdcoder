## Highlights

- Aligns module, data-model, naming, and API documentation with explicit domain ownership.
- Separates BirdCoder-owned application orchestration from dependency-owned IAM, Agents, IM, and Skills capabilities.
- Treats generated SDKs and their source OpenAPI documents as the only transport authority for dependency integrations.

## Notes

- BirdCoder does not own Skill packages, artifacts, capabilities, categories, assets, actions, or installations. Those resources are consumed from `sdkwork-skills` through its canonical App SDK.
- Application templates may keep stable installation identifiers for orchestration, but no dependency-owned record is copied into BirdCoder persistence.
