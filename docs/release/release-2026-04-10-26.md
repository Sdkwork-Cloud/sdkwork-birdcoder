## Highlights

- Materializes the shared desktop SQLite authority file into real direct provider tables for coding-session and representative app/admin resources, instead of leaving `kv_store` as the only persisted authority shape.
- Mirrors shared `table.sqlite.*` payload writes from desktop `local_store_set` and `local_store_delete` into those real tables, and replays older `kv_store` rows once on first open.
- Tightens Rust `coding-server` direct-table detection to require the full provider-table set, while keeping `kv_store` as an explicit bridge fallback instead of misclassifying partial SQLite files.
