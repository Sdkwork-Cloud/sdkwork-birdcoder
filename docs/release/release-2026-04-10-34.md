## Highlights

- Closes the first unified app/admin SDK consumer slice for default IDE catalog reads by freezing a shared typed client/transport contract on `server-api.ts`.
- Adds shared in-process and HTTP app/admin client transports, then routes default workspace/project catalog reads through that client boundary instead of direct repository reads.
- Preserves the current local write path and coding-session in-memory behavior, so this loop only closes read-side SDK/OpenAPI unification and does not overclaim host bootstrap or remote write closure.
