## Highlights

- Exposes governed release catalog reads through a dedicated `releaseService` on top of the shared generated app/admin facade.
- Propagates that service through default IDE services, shared contexts, and `useReleases()` so app consumers stop ending at the client layer only.
- Makes the closure executable in `check:release-flow`.

## Post-release operations

- Observation window: `0` minutes on `pending`.
- Stop-ship signals: `releaseService` missing from default IDE services or shared contexts; governed release catalog reads bypassing `BirdCoderAppAdminApiClient.listReleases()`; or any `check:release-flow` regression on the new contract.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-11-04 --release-assets-dir artifacts/release`.
- Re-issue path: `pnpm release:plan` -> release-service closure fix -> docs/release backwrite -> `pnpm release:finalize`.
- Writeback targets: `docs/架构/20-统一Rust-Coding-Server-API-协议标准.md`, `docs/架构/09-安装-部署-发布标准.md`, `docs/step/17-Coding-Server-Core-App-Admin-API与控制台实现.md`, `docs/step/17O-Default-IDE-Release-Service-Adoption-Lane.md`, `docs/prompts/反复执行Step指令.md`, `docs/release/releases.json`, and `docs/release/release-2026-04-11-04.md`.

## Notes

- This loop closes the first application-layer governed release read boundary only; it does not yet add typed write facades or broader core-read adoption into app consumers.
- PostgreSQL live smoke remains environment-blocked until a DSN-backed run produces a real `passed` report.
