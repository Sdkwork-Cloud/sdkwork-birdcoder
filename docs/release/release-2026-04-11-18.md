## Highlights

- Closes `17ZB` by introducing a dedicated `governance_policies` authority model and turning `admin.listPolicies` into a real Rust-host governance read across demo, legacy sqlite migration, and direct sqlite provider modes.
- Promotes `listPolicies()` into the shared generated app/admin facade instead of faking policy governance through deployment or audit truth.
- Adds `adminPolicyService` plus `loadAdminPolicies()` / `useAdminPolicies()` as the first reusable admin policy-facing consumer path and makes both part of release-flow governance.
