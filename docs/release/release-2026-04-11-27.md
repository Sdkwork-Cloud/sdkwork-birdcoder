## Highlights

- Re-baselines BirdCoder architecture so the in-repo product core now explicitly focuses on `Code` and `Studio`.
- Downgrades `Terminal` from local implementation lane to external-project integration boundary.
- Rewrites Step planning and loop prompt truth so external Terminal integration no longer blocks the main delivery path.

## Post-release operations

- Observation window: `0` minutes on `pending`.
- Stop-ship signals: any new doc or step plan that again treats Terminal as an in-repo implementation mainline, or any loop prompt that lets Step `08` block `Code / Studio`.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-11-27 --release-assets-dir artifacts/release`.
- Re-issue path: keep `Code / Studio` as mainline, restore external Terminal wording, rerun release-flow, governance-regression, and docs build, then `pnpm release:finalize`.
- Writeback targets: `docs/架构/02-架构标准与总体设计.md`, `docs/架构/03-模块规划与边界.md`, `docs/架构/15-工作台偏好-终端运行时-本地存储补充标准.md`, `docs/架构/16-终端主机会话-运行配置-本地存储标准.md`, `docs/step/08-terminal主机会话-cli集成-sqlite标准化.md`, `docs/step/README.md`, `docs/step/99-Step总执行矩阵与最短路径总表.md`, `docs/prompts/反复执行Step指令.md`, `docs/release/releases.json`, and `docs/release/release-2026-04-11-27.md`.
