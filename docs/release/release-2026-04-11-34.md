## Highlights

- Advances Step 06 `CP06-3` by extracting Code run-entry orchestration into a dedicated `useCodeRunEntryActions()` boundary.
- Freezes the new boundary with `code-run-entry-boundary-contract.test.mjs` and promotes it into both `lint` and `check:release-flow`.
- Keeps Code behavior unchanged while removing direct run-configuration persistence and launch-resolution ownership from `CodePage`.

## Post-release operations

- Observation window: `0` minutes on `pending`.
- Stop-ship signals: `CodePage` reintroduces `useProjectRunConfigurations()`, `resolveRunConfigurationTerminalLaunch()`, or `buildTerminalProfileBlockedMessage()`; the new contract drops out of `lint` or `check:release-flow`; run-entry launch logic drifts back into the page shell.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-11-34 --release-assets-dir artifacts/release`.
- Re-issue path: restore the dedicated run-entry hook, rerun the verification set, then refresh docs and release registry.
- Writeback targets: `packages/sdkwork-birdcoder-code/src/pages/CodePage.tsx`, `packages/sdkwork-birdcoder-code/src/pages/useCodeRunEntryActions.ts`, `scripts/code-run-entry-boundary-contract.test.mjs`, `scripts/terminal-cli-registry-contract.test.ts`, `scripts/run-config-request-contract.test.ts`, `package.json`, `docs/架构/03-模块规划与边界.md`, `docs/step/06-code视图-编辑器-文件系统重构.md`, `docs/prompts/反复执行Step指令.md`, `docs/release/release-2026-04-11-34.md`, `docs/release/releases.json`.

## Notes

- This release closes the run-entry-boundary slice only; `CP06-3` still needs editor/diff behavior-level regression proof.
- Governance regression contracts for terminal launch guard and run-config request now accept the dedicated run-entry hook as the valid Code-side consumer.
- Terminal remains an external integration boundary; this release does not reopen terminal runtime scope.
