## Highlights

- Defines the BirdCoder runtime assembly boundary for Prompt and AppTemplate flows.
- Keeps Skills execution outside BirdCoder: prompt assembly accepts `skills_context` derived from canonical installation references and does not implement a local Skill runtime.
- Enforces the runtime and SDK ownership boundaries through release-flow and governance regression checks.

## Current architecture boundary

- `assembleBirdCoderPromptRuntime()` composes BirdCoder prompt layers.
- `instantiateBirdCoderAppTemplateRuntime()` resolves BirdCoder-owned AppTemplate inputs.
- Skill discovery and installation use `@sdkwork/skills-app-sdk`; Skill package, artifact, capability, installation, and execution authority remain in `sdkwork-skills`.
- An installation request must select an immutable published `artifactId` and a workspace target explicitly.

## Release guardrails

- Stop shipment if a BirdCoder runtime starts resolving, copying, or executing Skill artifacts independently of `sdkwork-skills`.
- Stop shipment if an installation silently selects a latest artifact or if renderer-controlled identity fields reach the Skills API.
- Re-issue only after the canonical SDK, prompt composition, and AppTemplate contracts pass together.
