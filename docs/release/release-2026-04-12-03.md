## Highlights

- Adds the first provider and UnitOfWork-backed persistence slice for BirdCoder Prompt and AppTemplate evidence.
- Persists `prompt_run`, `prompt_evaluation`, and `app_template_instantiation` through shared repository contracts.
- Keeps Skill package, artifact, capability, and installation records outside BirdCoder persistence.

## Release guardrails

- Prompt and AppTemplate evidence must use the shared provider and UnitOfWork boundary.
- Evidence storage must not grow local Skills tables, copied Skill payloads, or sidecar state.
- PostgreSQL live verification requires a configured DSN and must be reported separately from static contract checks.
