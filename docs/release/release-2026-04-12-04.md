## Highlights

- Connects BirdCoder project services to the shared Prompt and AppTemplate evidence repositories.
- Keeps evidence writes on the provider and UnitOfWork path instead of service-owned sidecar state.
- Preserves `sdkwork-skills` as the only authority for reusable Skill resources and installations.

## Release guardrails

- Service consumers must not bypass the shared provider and UnitOfWork repositories.
- AppTemplate evidence may reference canonical Skill installation IDs but must not persist dependency-owned Skill records.
- PostgreSQL live verification requires a configured DSN and must be reported separately from static contract checks.
