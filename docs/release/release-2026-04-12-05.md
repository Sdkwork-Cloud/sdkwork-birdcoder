## Highlights

- Connects coding-server Prompt and AppTemplate evidence writes to the shared provider and UnitOfWork boundary.
- Writes `prompt_run` and `prompt_evaluation` in the same transaction as their BirdCoder runtime evidence.
- Keeps all reusable Skill data and installation authority in `sdkwork-skills`.

## Release guardrails

- Coding-server evidence writes must remain atomic with the associated BirdCoder runtime transaction.
- Evidence records must not become a projection or copy of Skill packages, artifacts, capabilities, or installations.
- PostgreSQL live verification requires a configured DSN and must be reported separately from static contract checks.
