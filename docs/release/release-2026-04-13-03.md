## Highlights

- Adds the repo-governed PostgreSQL runtime dependency lane by resolving `pg` through the workspace catalog, so DSN-backed live smoke can execute in this workspace instead of stopping at `missing_postgresql_driver`.
- Fixes the PostgreSQL live-smoke cleanup path so DSN-configured connection failures now return structured `failed` reports instead of being replaced by uncaught exceptions during provider shutdown.
- Rewrites the active release and Step docs to keep the environment gate truthful: missing DSN or driver remains `blocked`, reachable smoke success remains `passed`, and DSN-backed connectivity failures remain auditable `failed`.
