## Highlights

- Hardens `scripts/quality-gate-execution-report.mjs` so the CLI no longer depends on a module-level top-level `await`, and makes the Windows tier-runner contract deterministic by injecting `spawnSync` during tests instead of relying on host child-process permissions.
- Hardens `scripts/run-postgresql-live-smoke.ts` to use the same direct-entry guarded, non-top-level-await CLI pattern, preserving the existing `passed -> 0`, `blocked -> 2`, and `failed -> 1` executable status contract.
- Freezes both entrypoint protections in contract coverage so future regressions reopen fast in `quality-gate-execution-report.test.mjs` and `postgresql-live-smoke-contract.test.ts`.
