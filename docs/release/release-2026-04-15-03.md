## Highlights

- Closes the remaining package-script import-side-effect drift by hardening `check:arch`, `check:sdkwork-birdcoder-structure`, and `generate:rust-host-engine-catalog` behind explicit callable functions plus direct-entry CLI guards instead of executing immediately on module import.
- Adds a dedicated package-script entrypoint contract and wires it into `check:release-flow`, so future regressions reopen if these governed `package.json` entrypoints lose import safety or executable launch discipline.
- Regenerates the Rust host shared engine catalog to the current coding-server truth, including the promoted `sdk-stream` transport metadata now exposed by the TypeScript engine/kernel layer.

## Notes

- The new entrypoint contract protects specifically the governed package scripts that were still import-unsafe in this workspace: `check:arch`, `check:sdkwork-birdcoder-structure`, and `generate:rust-host-engine-catalog`.
- The regenerated Rust host engine catalog now records `sdk-stream` in the shared engine/model transport metadata, keeping the Rust host artifact aligned with the promoted coding-server engine truth consumed from TypeScript.

- Observation goal: keep all governed package-script entrypoints import-safe and keep the Rust host shared engine catalog aligned with promoted coding-server engine truth.
