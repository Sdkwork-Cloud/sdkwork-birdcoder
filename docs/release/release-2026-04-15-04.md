## Highlights

- Closes the remaining package-script contract discoverability drift by exposing `check:package-script-entrypoints` as a first-class root verification command instead of leaving the new gate reachable only transitively through `check:release-flow`.
- Extends both the repository structure check and the top-level architecture contract so `scripts/package-script-entrypoints-contract.test.mjs` is now frozen as required repo topology, not just an opportunistic test file.
- Keeps the newly hardened package-script entrypoint contract inside the release-flow path while also making it individually rerunnable for focused verification and future incident triage.

- `node scripts/check-sdkwork-birdcoder-structure.mjs`
- `pnpm.cmd run check:package-script-entrypoints`
- `pnpm.cmd run check:release-flow`

## Notes

- This release does not change the already-added package script entrypoint contract behavior; it changes governance discoverability and repository topology so the contract cannot disappear silently from the workspace standard.
- The new root script shortens focused reruns during future debugging: engineers can now validate guarded package-script entrypoints directly without paying the full release-flow cost.
- The previously documented Windows child-process blocker remains unchanged and is still represented as structured `blocked` output in governance and quality execution reports.

## Post-release operations

- Observation goal: keep the package-script entrypoint contract discoverable as a direct root command and preserved inside the required repository topology.
- Observation window: after any edit to `package.json`, `scripts/check-sdkwork-birdcoder-structure.mjs`, `scripts/sdkwork-birdcoder-architecture-contract.test.mjs`, or `scripts/package-script-entrypoints-contract.test.mjs`, rerun the verification commands listed above plus `pnpm.cmd run check:release-flow`.
- Stop-ship signals: the root manifest drops `check:package-script-entrypoints`; structure or architecture contracts stop requiring `scripts/package-script-entrypoints-contract.test.mjs`; or release-flow no longer executes the entrypoint contract.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-15-04 --release-assets-dir artifacts/release`.
- Re-issue path: restore the root package-script verification command and topology assertions -> rerun `node scripts/sdkwork-birdcoder-architecture-contract.test.mjs`, `node scripts/check-sdkwork-birdcoder-structure.mjs`, `pnpm.cmd run check:package-script-entrypoints`, and `pnpm.cmd run check:release-flow`.
- Writeback targets: `package.json`, `scripts/check-sdkwork-birdcoder-structure.mjs`, `scripts/sdkwork-birdcoder-architecture-contract.test.mjs`, `docs/release/release-2026-04-15-04.md`, and `docs/release/releases.json`.
