## Highlights

- Closes the remaining root package-script topology drift by teaching `check:sdkwork-birdcoder-structure` to derive repo-local `scripts/*` targets directly from the root `package.json` manifest instead of depending only on a hand-maintained required-path list.
- Adds a dedicated `check:sdkwork-birdcoder-structure-contract` regression that builds a temporary workspace, proves a missing `package.json` script target is rejected, and verifies the error stays actionable by naming the owning root script.
- Wires that new structure-behavior contract into `check:release-flow` and freezes its presence in both structure and top-level architecture governance so future manifest-to-filesystem drift reopens immediately.

## Notes

- The structure check now treats the root manifest as the authoritative source for repo-local script entrypoints, which removes the prior gap where a new or renamed package script could escape structure governance until someone manually updated `requiredPaths`.
- The new behavior contract verifies structure-governance semantics, not just file presence or source text shape, by running the checker against a synthetic workspace that intentionally references a missing script file.
- The previously documented Windows child-process blocker remains unchanged and continues to surface as structured `blocked` output in governance and quality execution reports, not as a reopened repository regression.

## Post-release operations

- Observation goal: keep root `package.json` script entrypoints and on-disk repo-local script files in permanent lockstep.
- Observation window: after any edit to `package.json`, `scripts/check-sdkwork-birdcoder-structure.mjs`, `scripts/check-sdkwork-birdcoder-structure-contract.test.mjs`, `scripts/run-release-flow-check.mjs`, or `scripts/sdkwork-birdcoder-architecture-contract.test.mjs`, rerun the verification commands listed above plus `pnpm.cmd run check:release-flow`.
- Stop-ship signals: `check:sdkwork-birdcoder-structure` stops failing when a root package script references a missing repo-local file; the root manifest drops `check:sdkwork-birdcoder-structure-contract`; or release-flow no longer executes the structure behavior contract.
