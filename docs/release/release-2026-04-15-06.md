## Highlights

- Closes the remaining governance-regression topology blind spot by teaching `runGovernanceRegressionReport` to preflight each governed check's repo-local `scriptPath`, repo-local command target, and `pnpm run` root-script binding before execution.
- Prevents Windows `spawn EPERM` host blockers from masking repository drift by failing invalid governance check definitions as `failed` while preserving the existing structured `blocked` outcome when topology is valid and only the command runner is unavailable.
- Promotes `scripts/governance-regression-report.test.mjs` into `check:release-flow`, so release gating now exercises the governance runner contract directly instead of relying on broader lint lanes to catch regressions later.

## Notes

- The new governance topology preflight is intentionally repo-focused and environment-independent: it validates missing files and missing root package script bindings before any child-process launch is attempted.
- On this host, the direct governance regression report remains `blocked` because `web-bundle-budget` still requires child-process access to the Windows command runner; the new preflight does not change that contract when repository topology is healthy.
- The new runtime behavior is also frozen by test with a synthetic temporary workspace, proving invalid governance check topology is surfaced as `failed` and skipped from execution instead of being hidden behind unrelated later blockers.

## Post-release operations

- Observation goal: keep governance regression check definitions executable, discoverable, and impossible to mask behind host-level command-runner blockers.
- Observation window: after any edit to `scripts/governance-regression-report.mjs`, `scripts/governance-regression-report.test.mjs`, `scripts/run-release-flow-check.mjs`, or `scripts/run-release-flow-check.test.mjs`, rerun the verification commands listed above plus inspect `artifacts/governance/governance-regression-report.json`.
- Stop-ship signals: governance regression checks can reference missing repo-local files or missing root package scripts without failing the contract; `check:release-flow` stops executing `scripts/governance-regression-report.test.mjs`; or a topology-healthy direct governance report stops producing the documented `blocked` outcome on this host.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-15-06 --release-assets-dir artifacts/release`.
- Re-issue path: restore governance topology preflight and release-flow coverage -> rerun `node scripts/governance-regression-report.test.mjs`, `node scripts/run-release-flow-check.test.mjs`, `pnpm.cmd run check:release-flow`, and `node scripts/governance-regression-report.mjs`.
- Writeback targets: `scripts/governance-regression-report.mjs`, `scripts/governance-regression-report.test.mjs`, `scripts/run-release-flow-check.mjs`, `scripts/run-release-flow-check.test.mjs`, `docs/release/release-2026-04-15-06.md`, and `docs/release/releases.json`.
