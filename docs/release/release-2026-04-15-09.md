## Notes

- Before this round, the repo-tracked canonical release asset still preserved the old workflow-only quality summary, so it omitted `manifestBoundTiers`, omitted the packaged Step 18 governance set, and rendered release notes that could not surface the current Step 12 topology truth.
- The new contract freezes only stable packaged truth: tier commands, workflow/manifest binding coverage, packaged `qualityEvidence` topology summary, and release-note rendering. That keeps the contract valuable without making host-only blocker drift the reason a canonical artifact suddenly looks stale.
- The refreshed canonical asset now reports `workflowBoundTiers=3`, `manifestBoundTiers=3`, full `releaseGovernanceCheckIds`, and a packaged loop scoreboard whose lowest score item is the remaining host blocker `commercial_readiness`, not a stale topology artifact.

## Post-release operations

- Observation goal: keep repo-tracked canonical release assets impossible to drift behind the current quality-evidence contract without tripping `check:release-flow`.
- Observation window: after any edit to `scripts/quality-gate-matrix-report.mjs`, `scripts/release/quality-gate-release-evidence.mjs`, `scripts/release/render-release-notes.mjs`, or canonical release asset generation paths, rerun the verification commands above and inspect `artifacts/release-openapi-canonical`.
- Stop-ship signals: canonical `quality-gate-matrix-report.json` drops manifest-bound topology truth; canonical `release-manifest.json` loses the packaged `qualityEvidence` summary fields; canonical `release-notes.md` stops surfacing manifest-bound tiers or packaged governance checks; or `check:release-flow` stops executing the canonical release-evidence freshness contract.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-15-09 --release-assets-dir artifacts/release-openapi-canonical`.
- Re-issue path: restore canonical release evidence freshness contract -> rerun canonical finalize/smoke/release-note generation -> rerun `node scripts/release-openapi-canonical-quality-evidence-contract.test.mjs` and `pnpm.cmd run check:release-flow`.
- Writeback targets: `scripts/release-openapi-canonical-quality-evidence-contract.test.mjs`, `scripts/run-release-flow-check.mjs`, `scripts/run-release-flow-check.test.mjs`, `artifacts/release-openapi-canonical/release-manifest.json`, `artifacts/release-openapi-canonical/quality/quality-gate-matrix-report.json`, `artifacts/release-openapi-canonical/finalized-release-smoke-report.json`, `artifacts/release-openapi-canonical/release-notes.md`, `artifacts/release-openapi-canonical/SHA256SUMS.txt`, `docs/release/release-2026-04-15-09.md`, and `docs/release/releases.json`.
