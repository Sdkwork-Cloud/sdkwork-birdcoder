## Notes

- Before this round, packaged `qualityEvidence` kept `workflowBoundTiers` and `manifestBoundTiers` but dropped `missingWorkflowBindings` and `missingManifestBindings`, so finalized assets could say topology drift existed without preserving which tier ids were out of contract.
- That was a commercial delivery weakness: packaged evidence was no longer sufficient for audit, release-note diagnosis, or stop-ship review without reopening the workspace report.
- The new contract keeps host truth intact. The current blocker is still the environment-scoped `vite-host-build-preflight` diagnostic, not a repository regression.

## Post-release operations

- Observation goal: keep finalized release evidence self-diagnosing so topology drift remains actionable from packaged artifacts alone.
- Observation window: after any edit to `scripts/quality-gate-matrix-report.mjs`, `scripts/release/quality-gate-release-evidence.mjs`, `scripts/release/render-release-notes.mjs`, or canonical release-asset generation paths, rerun the verification commands above and inspect `artifacts/release-openapi-canonical`.
- Stop-ship signals: finalized `qualityEvidence` loses `missingWorkflowBindings` or `missingManifestBindings`; release notes stop rendering missing tier ids when drift exists; canonical packaged artifacts fall behind the new summary contract; or `check:release-flow` stops enforcing the canonical freshness test.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-15-10 --release-assets-dir artifacts/release-openapi-canonical`.
