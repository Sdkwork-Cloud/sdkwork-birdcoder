## Highlights

- Closes the remaining packaged runtime-evidence regression by making release finalization automatically preserve an existing or workspace `quality-gate-execution-report.json` instead of silently dropping execution-state truth unless a manual flag is passed.
- Closes the remaining checksum-integrity gap by making `render-release-notes.mjs` refresh `SHA256SUMS.txt` whenever it writes `release-notes.md` inside a finalized release asset directory.
- Refreshes the canonical OpenAPI release asset so packaged manifest, finalized smoke, rendered release notes, and checksum inventory now agree on the same runtime-blocked quality verdict.

## Notes

- Before this round, repeating `finalize` without `--quality-execution-report-path` could silently strip runtime execution truth from packaged `qualityEvidence`, even though the workspace already had `artifacts/quality/quality-gate-execution-report.json`.
- The same release lane also depended on operator sequencing for checksum correctness: rendering release notes after finalize could leave `SHA256SUMS.txt` stale unless someone manually regenerated it.
- Both issues are release-integrity defects, not feature work. They affect auditability, blocker truthfulness, and the reliability of downstream commercial publication tooling.

## Post-release operations

- Observation goal: keep finalized release assets self-consistent so runtime quality truth and checksum truth survive repeated local finalize and note-render flows.
- Observation window: after any edit to `scripts/release/quality-gate-release-evidence.mjs`, `scripts/release/render-release-notes.mjs`, release-asset finalization paths, or canonical release-asset regeneration commands, rerun the verification commands above and inspect `artifacts/release-openapi-canonical`.
- Stop-ship signals: finalized `qualityEvidence` loses execution archive or runtime status fields; canonical release notes stop surfacing runtime execution evidence; `SHA256SUMS.txt` stops matching `release-notes.md`; or `check:release-flow` stops enforcing the canonical packaged runtime-evidence contract.
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-04-15-11 --release-assets-dir artifacts/release-openapi-canonical`.
- Re-issue path: restore packaged runtime execution evidence preservation -> rerender release notes inside the target asset directory -> confirm `SHA256SUMS.txt` refresh -> rerun the verification commands above.
