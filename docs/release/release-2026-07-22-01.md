# SDKWork BirdCoder Pre-Launch Release State

Status: pending
Updated: 2026-07-22

This is the only pre-launch release record. It describes the current target
architecture and does not preserve superseded implementation history.

## Current Scope

- BirdCoder owns the coding-workbench bounded context, ten `studio_*` tables,
  39 App API operations, and no Backend API or Open API operations.
- Agents owns AI projects, sessions, turns, session items, interactions,
  runtime bindings, artifacts, checkpoints, and provider execution.
- Skills owns skill packages, marketplaces, artifacts, capabilities,
  installations, assets, actions, APIs, SDKs, and persistence.
- IM owns human conversations, messages, members, and read cursors. AI assistant
  transcript items are Agents Session Items and are not IM messages.
- Persistent projections, synchronized shadow tables, dual writes, compatibility
  facades, raw HTTP consumers, manual authentication headers, and local SDK forks
  are prohibited.

## Verification State

- Domain ownership, database parity, API operation, response envelope,
  pagination, route collision, layering, SDK consumer, package governance, PC,
  H5, Flutter, Rust gateway, desktop host, and documentation gates are part of
  the completed cutover evidence.
- Release packaging, checksum, SBOM, rollback, and rehearsal contracts are
  governed and remain fail-closed.
- Formal promotion is blocked only by
  `signed-production-artifact-evidence-missing`.

## Post-release operations

- Observation goal: keep the single-owner domain boundaries, SDK-only dependency
  consumption, and single Rust application ingress aligned through promotion.
- Observation window: from completion of the architecture cutover until the
  first signed production candidate completes the configured rollout window.
- Stop-ship signals: any ownership violation, schema or SDK drift, or failed
  production gate blocks promotion. The current blocker list contains exactly
  one entry.
- Release kind: `pre-launch`
- Rollout stage: `not-started`
- Formal or GA status: `blocked`
- Machine stop-ship signals: signed-production-artifact-evidence-missing
- Rollback entry: `pnpm release:rollback:plan -- --release-tag release-2026-07-22-01 --release-assets-dir artifacts/release`
- Re-issue path: complete all release gates, generate immutable signed artifacts,
  verify checksums and attestations, then update this registry and note together.
- Writeback targets: `docs/release/releases.json`,
  `docs/release/release-2026-07-22-01.md`, and the canonical release evidence
  bundle.
