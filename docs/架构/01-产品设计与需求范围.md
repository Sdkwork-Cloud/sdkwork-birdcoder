# 01-产品设计与需求范围

## 1. 产品定位

BirdCoder 是面向个人开发者、团队研发、平台工程团队的超级 AI IDE。它不是单一聊天界面，而是统一承载代码理解、生成、改写、构建、运行、预览、测试、发布的开发工作台。

## 2. 核心角色

- 开发者：编写代码、调试、构建、预览、测试、发布。
- 团队负责人：管理模板、规范、共享环境、发布策略。
- 平台管理员：配置引擎、权限、部署环境、审计与成本控制。

## 3. 当前产品基线

- 主界面已具备 `code / studio / terminal / skills / templates / user / settings` 多页签能力。
- 已具备工作区、项目、线程、消息、文件树、终端、多配置项等基础交互。
- 已暴露代码开发引擎、审批策略、沙箱设置、MCP、Git、环境、worktree 等设置入口。
- 已具备桌面窗口控制、打开本地文件夹、终端 profile、聊天流式输出和 tool call 映射。

## 4. 目标需求

- 统一接入 Claude Code、Codex、OpenCode 等 Code Engine。
- 统一会话、上下文、工具调用、补丁、命令、审批、审计标准。
- 支持代码、终端、预览、模拟器、测试、发布的完整开发闭环。
- 支持 Web、Desktop、Server、Docker、Kubernetes、多 OS、多 CPU 架构打包。
- 支持团队级治理，包括权限、规则、技能、MCP、发布证据、审计与成本控制。

## 5. 非功能目标

- 高内聚：Kernel 聚合通用能力，业务模块只处理自身职责。
- 低耦合：Engine、Host、Preview、Build、Release 全部通过契约接入。
- 高性能：启动快、首响应快、增量构建快、预览刷新快。
- 高安全：最小权限、密钥隔离、审计闭环、供应链可追溯。
- 易维护：包边界清晰、可替换、可测试、可观测。

## 6. 范围边界

本目录只复制高标准架构方法，不复制参考项目业务语义。BirdCoder 保持“超级 AI IDE”形态不变，但在宿主、内核、发布、部署、规则、治理层面与行业最佳实践对齐。

## 7. 评估标准

- 需求覆盖率：核心开发链路覆盖率达到 100%。
- 跨引擎一致性：主流程能力不因引擎切换而产生交互断层。
- 跨宿主一致性：Web、Desktop、Server 主界面与主流程保持同一语义。
- 发布完备性：本地、容器、K8s、GitHub Release 全链路可执行。
